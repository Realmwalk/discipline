# Discipline Docs

Project-local documentation for Discipline (the OSS framework). The public-facing landing-page README is at the repo root.

## Documentation map

- [`HANDOFF.md`](HANDOFF.md) — first-read for AI agents working on Discipline development.
- [`AGENTS.md`](AGENTS.md) — Discipline's own AI playbook with Local Overrides on the framework defaults.
- [`DECISIONS.md`](DECISIONS.md) — durable architectural / product decisions.
- [`CHANGELOG.md`](CHANGELOG.md) — chronological log of shipped work.
- [`HARNESS_INTEGRATION.md`](HARNESS_INTEGRATION.md) — how Discipline binds to specific AI coding harnesses (Claude Code, OpenCode, Codex, Cursor, …) via thin per-harness adapters.

## Eat-your-own-dogfood

Discipline's own docs use Discipline's templates. If we can't sustain this project using the framework we ship, the framework is wrong — that's the meta-validation principle.
